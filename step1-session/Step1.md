# Step1 (Session) 목표
- 순수 세션 로그인
- 인증 구조 학습 + 기본기 완성 단계

# 기본 로그인
- ID/PW 로그인
- 세션 유지 (시간: 30분)
- 로그인/로그아웃
- 비밀번호 변경
- 유효성 검사

# 로그인 구조 (서버 기준/흐름)
POST /login
→ email, password 받음
→ DB 조회
→ 비밀번호 검증
→ 세션 생성
→ JSESSIONID 쿠키 반환
👉 요청 → 쿠키 자동 포함 → 서버에서 세션으로 사용자 식별

# 로그인 로직
- RuntimeException 오류 메시지 통일(보안)
- 존재 여부 노출 금지

# 필수 기능

## 리빌드
- 인터셉터 기반 인증 체크
- 세션 만료 처리
- 비밀번호 update 정상화
- 예외 구조 통일
- 유지보수 가능한 구조

## UX 핵심 포인트 정리
1. 로그인 실패 -> redirect + error 메시지
2. input 유지 가능 (확장 가능)
3. 버튼 중복 클릭 방지
4. 메시지 통일 (보안)
5. 세션 만료 -> 자동 로그인 페이지 이동

## 테스트 흐름
1. 회원가입
→ /signup
2. 로그인
→ /login
3. 세션 생성 확인
→ JSESSIONID 쿠키
4. 보호 페이지 접근
→ 인터셉터 통과
5. 로그아웃
→ 세션 제거
6. 다시 접근
→ /login 리다이렉트
